## 2. 测试用例详情

### 2.1 认证模块 (Auth Module)

#### TC-AUTH-001: 微信登录 - 成功场景
**测试目的**: 验证微信登录流程正常
**前置条件**: 提供有效的微信code
**操作步骤**:
1. 发送POST请求到 `/api/v1/auth/wx-login`
2. 请求体包含有效的code
**预期结果**:
- HTTP状态码: 200
- 返回包含user、accessToken、refreshToken的响应
- 数据库中创建新用户记录

#### TC-AUTH-002: 微信登录 - 空code
**测试目的**: 验证空code的处理
**前置条件**: code参数为空
**操作步骤**:
1. 发送POST请求到 `/api/v1/auth/wx-login`
2. 请求体code为空字符串
**预期结果**:
- HTTP状态码: 401
- 返回错误信息: "微信登录凭证不能为空"

#### TC-AUTH-003: 微信登录 - 无效code
**测试目的**: 验证无效code的处理
**前置条件**: code参数无效
**操作步骤**:
1. 发送POST请求到 `/api/v1/auth/wx-login`
2. 请求体code为无效值
**预期结果**:
- HTTP状态码: 401
- 返回微信登录失败错误

#### TC-AUTH-004: Token刷新 - 成功场景
**测试目的**: 验证Token刷新功能
**前置条件**: 提供有效的refreshToken
**操作步骤**:
1. 先执行登录获取refreshToken
2. 发送POST请求到 `/api/v1/auth/refresh`
3. 请求头包含Authorization: Bearer {refreshToken}
**预期结果**:
- HTTP状态码: 200
- 返回新的accessToken和refreshToken

#### TC-AUTH-005: Token刷新 - 无效Token
**测试目的**: 验证无效Token的处理
**前置条件**: 提供无效的refreshToken
**操作步骤**:
1. 发送POST请求到 `/api/v1/auth/refresh`
2. 请求头包含无效的Authorization
**预期结果**:
- HTTP状态码: 401
- 返回错误信息: "令牌无效或已过期"

#### TC-AUTH-006: Token刷新 - 空Token
**测试目的**: 验证空Token的处理
**前置条件**: Authorization头为空
**操作步骤**:
1. 发送POST请求到 `/api/v1/auth/refresh`
2. 不提供Authorization头
**预期结果**:
- HTTP状态码: 401
- 返回错误信息: "刷新令牌不能为空"

## 3. 边界条件测试

### 3.1 输入边界测试

| 测试项 | 测试值 | 预期结果 |
|--------|--------|----------|
| page参数 | 0 | 视为1或返回错误 |
| page参数 | -1 | 返回错误 |
| page参数 | 999999 | 返回空数组 |
| limit参数 | 0 | 视为默认值或返回错误 |
| limit参数 | -1 | 返回错误 |
| limit参数 | 101 | 限制为最大值100 |
| limit参数 | 1000 | 限制为最大值100 |
| 空字符串参数 | "" | 忽略该参数 |
| 超长字符串 | 1000字符 | 正常处理或截断 |

### 3.2 数据边界测试

| 测试项 | 测试场景 | 预期结果 |
|--------|----------|----------|
| 空数据库 | 无任何数据 | 返回空数组，不报错 |
| 单条数据 | 只有一条记录 | 正确返回单条数据 |
| 大量数据 | 10000条记录 | 分页正常，性能可接受 |
| 特殊字符 | 名称含特殊字符 | 正确处理，不报错 |
| 超长文本 | 描述字段超长 | 正常存储和返回 |

### 5.1 认证安全
- Token过期后无法访问受保护资源
- 无效Token返回401错误
- Token不能伪造

### 5.2 输入安全
- SQL注入防护
- XSS攻击防护
- 特殊字符处理

### 5.3 访问控制
- 未认证用户无法访问受保护接口
- 用户只能访问自己的数据
