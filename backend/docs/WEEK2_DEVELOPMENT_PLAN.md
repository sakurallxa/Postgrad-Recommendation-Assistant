## 二、详细实施计划

### 2.1 微信登录真实API调用

**目标**: 实现真实的微信登录流程，调用微信官方API

**实现步骤**:
1. 更新AuthService，实现真实的微信API调用
2. 添加错误处理和重试机制
3. 实现用户信息缓存
4. 添加登录日志记录

**接口变更**:
```typescript
// POST /api/v1/auth/wx-login
// 请求: { code: string }
// 响应: { user, accessToken, refreshToken, expiresIn }
```

### 2.2 院校详情和筛选功能

**目标**: 完善院校查询功能，支持更多筛选条件

**实现步骤**:
1. 实现院校详情接口（包含专业列表）
2. 添加关键词搜索功能
3. 支持多条件组合筛选
4. 添加院校排序功能

**新增接口**:
```typescript
// GET /api/v1/universities/:id
// 获取院校详情

// GET /api/v1/universities?keyword=清华&region=北京&level=985
// 支持关键词搜索
```

### 2.3 用户偏好设置功能

**目标**: 实现用户关注院校和专业的功能

**实现步骤**:
1. 创建User模块
2. 实现用户偏好设置接口
3. 添加用户选择验证
4. 实现个性化推荐基础

**新增接口**:
```typescript
// GET /api/v1/user/profile
// 获取用户信息

// GET /api/v1/user/selection
// 获取用户选择

// PUT /api/v1/user/selection
// 更新用户选择
```

### 2.4 API限流和日志中间件

**目标**: 添加API限流保护和请求日志记录

**实现步骤**:
1. 实现基于IP的限流中间件
2. 添加请求响应日志中间件
3. 实现API调用统计
4. 添加异常告警机制

**配置**:
```typescript
// 限流配置
RATE_LIMIT_WINDOW = 60000;  // 1分钟
RATE_LIMIT_MAX = 100;       // 最多100请求
```

## 三、技术规范

### 3.1 代码规范

- 所有新功能必须包含单元测试
- 接口必须添加Swagger文档
- 错误处理必须使用统一异常过滤器
- 数据库操作必须使用Prisma事务

### 3.2 测试要求

- 单元测试覆盖率 >= 80%
- 集成测试覆盖所有API端点
- 性能测试：API响应时间 < 200ms

### 3.3 文档要求

- 更新API文档
- 编写功能使用说明
- 记录配置变更
