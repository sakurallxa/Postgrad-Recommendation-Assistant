# 保研信息助手小程序 PRD

**版本**: v1.0  
**日期**: 2026-02-24  
**状态**: 待评审

---

## 1. Project DNA

为保研学生提供目标院校夏令营信息的精准聚合与截止日期智能提醒服务。

---

## 2. First Action Path

用户选择目标院校/专业 → 查看结构化夏令营信息 → 设置截止日期提醒

---

## 3. Technical Constraints

### 3.1 核心数据模型 (Data Model)

```
University {
  id: string          // 院校唯一标识
  name: string        // 院校名称
  logo: string        // 院校logo URL
}

Major {
  id: string          // 专业唯一标识
  name: string        // 专业名称
  category: string    // 学科门类
}

CampInfo {
  id: string              // 夏令营信息唯一标识
  universityId: string    // 关联院校ID
  majorId: string         // 关联专业ID
  title: string           // 夏令营标题
  sourceUrl: string       // 原始公告URL
  publishDate: date       // 发布日期
  deadline: date          // 报名截止日期
  startDate: date         // 夏令营开始日期
  endDate: date           // 夏令营结束日期
  requirements: string    // 申请要求(结构化JSON)
  materials: string[]     // 所需材料清单
  process: string[]       // 报名流程步骤
  status: enum            // 状态: [draft/published/expired]
}

UserSelection {
  userId: string          // 用户ID
  universityIds: string[] // 关注的院校列表
  majorIds: string[]      // 关注的专业列表
}

Reminder {
  id: string              // 提醒ID
  userId: string          // 用户ID
  campId: string          // 关联夏令营ID
  remindTime: date        // 提醒时间
  status: enum            // 状态: [pending/sent/expired]
}
```

### 3.2 状态流转 (State Machine)

**夏令营信息状态**:
```
draft(草稿) → published(已发布) → expired(已过期)
```

**提醒状态**:
```
pending(待发送) → sent(已发送) → expired(已过期)
```

## 4. Feature Spec

### 4.1 院校/专业选择模块

**功能描述**: 用户可按院校名称或专业名称进行筛选，支持多选组合。

**交互逻辑**:
1. 首页展示院校列表，支持按首字母/地区分类
2. 点击院校进入专业选择页
3. 用户可同时选择多个院校和多个专业
4. 选择结果持久化存储，下次进入自动恢复

**接口定义**:
- `GET /universities` - 获取院校列表
- `GET /majors?universityId={id}` - 获取某院校下的专业列表
- `POST /user/selections` - 保存用户选择

### 4.2 夏令营信息聚合模块

**功能描述**: 基于用户选择的院校和专业，聚合展示相关夏令营信息。

**展示字段**:
- 院校名称 + logo
- 夏令营标题
- 报名截止日期(倒计时高亮)
- 夏令营举办时间
- 申请要求摘要

**详情页结构化展示**:
- 基本信息: 时间、地点、主办方
- 申请条件: 学历要求、成绩要求、英语要求
- 所需材料: 清单式展示，支持一键复制
- 报名流程: 步骤化展示
- 原文链接: 跳转至官网原文

**接口定义**:
- `GET /camps?universityIds={ids}&majorIds={ids}` - 获取筛选后的夏令营列表
- `GET /camps/{id}` - 获取夏令营详情

### 4.3 截止日期提醒模块

**功能描述**: 用户可为关注的夏令营设置微信提醒，在截止日期前收到通知。

**提醒规则**:
- 默认提醒时间: 截止前3天、截止前1天、截止当天
- 用户可自定义提醒时间
- 支持订阅消息推送(微信小程序订阅消息能力)

**交互逻辑**:
1. 在夏令营详情页点击"设置提醒"
2. 弹出时间选择器，用户选择提醒时间
3. 调用微信订阅消息接口，获取用户授权
4. 后台定时任务扫描待发送提醒，触发推送

**接口定义**:
- `POST /reminders` - 创建提醒
- `DELETE /reminders/{id}` - 取消提醒
- `GET /reminders?userId={id}` - 获取用户提醒列表

## 6. 技术架构建议

### 6.1 前端
- 微信小程序原生开发 / Taro跨端框架
- 状态管理: MobX / Redux

### 6.2 后端
- Node.js + Express / Python + FastAPI
- 数据库: MySQL(关系数据) + Redis(缓存)
- 定时任务: Node-cron / Celery

### 6.3 数据采集
- 爬虫服务: Puppeteer / Scrapy
- 数据清洗: 结构化提取关键信息
- 更新频率: 每日定时爬取

### 8.3 差异化竞争优势分析

#### 优势一：极致聚焦，降低认知负担

| 维度 | 竞品现状 | 本项目优势 |
|------|----------|------------|
| 功能范围 | 大而全，信息过载 | 精准聚焦夏令营信息聚合+提醒 |
| 用户路径 | 多入口、多功能分散注意力 | 单一路径：筛选→查看→提醒 |
| 学习成本 | 需要探索多个功能模块 | 零学习成本，开箱即用 |

**价值主张**: "只做一件事，做到极致" —— 帮助保研学生高效获取夏令营信息，不错过任何截止日期。

#### 优势二：信息结构化深度优于竞品

| 维度 | 竞品现状 | 本项目优势 |
|------|----------|------------|
| 信息展示 | 多为原文转载或简单摘要 | 深度结构化：申请条件、材料清单、流程步骤 |
| 材料提取 | 用户需自行阅读原文识别 | 自动提取所需材料，清单式展示 |
| 流程呈现 | 文字堆砌 | 步骤化、时间线可视化 |

**价值主张**: "让信息自己说话" —— 用户无需逐字阅读公告原文，关键信息一目了然。

#### 优势三：微信原生提醒体验

| 维度 | 竞品现状 | 本项目优势 |
|------|----------|------------|
| 提醒方式 | APP内推送/短信/需下载APP | 微信订阅消息，无需额外APP |
| 提醒触达率 | APP卸载后失效 | 微信生态，触达率极高 |
| 用户习惯 | 需要主动打开APP查看 | 微信消息直达，符合用户习惯 |

**价值主张**: "提醒真正到达用户" —— 基于微信生态的订阅消息，确保关键节点不遗漏。

#### 优势四：零门槛使用

| 维度 | 竞品现状 | 本项目优势 |
|------|----------|------------|
| 注册门槛 | 需注册账号/手机号 | 微信一键授权登录 |
| 付费门槛 | 核心功能需付费 | 完全免费使用 |
| 下载门槛 | 需下载APP | 小程序即开即用 |

**价值主张**: "3秒开始使用" —— 无需下载、无需注册、无需付费。

### 9.3 核心痛点覆盖分析

#### 痛点一：信息分散，收集成本高

| 维度 | 分析 |
|------|------|
| **用户原话** | "保研的信息由各个学校官网学院官网发布，需要一个个收集" |
| **痛点本质** | 信息源分散(数百所院校官网)、检索效率低、信息更新追踪难 |
| **本项目解决方案** | 自动聚合目标院校夏令营信息，用户只需选择关注的院校/专业 |
| **覆盖程度** | ✅ **完全覆盖** |
| **体验提升** | 从"逐个官网查看"到"一站式聚合查看"，效率提升10倍+ |

#### 痛点二：关键信息识别提取困难

| 维度 | 分析 |
|------|------|
| **用户原话** | "里面的关键内容信息比如所需材料，流程，截止时间等需要自己识别抓取整理" |
| **痛点本质** | 公告格式不统一、关键信息散落在长文中、人工提取易遗漏 |
| **本项目解决方案** | 结构化展示：申请条件、材料清单、流程步骤、截止时间 |
| **覆盖程度** | ✅ **完全覆盖** |
| **体验提升** | 从"逐字阅读原文"到"关键信息一目了然"，信息获取效率提升5倍+ |

#### 痛点三：截止日期无有效提醒

| 维度 | 分析 |
|------|------|
| **用户原话** | "截止时间没有有效提醒" |
| **痛点本质** | 人工记录易遗漏、日历提醒需手动设置、多院校管理混乱 |
| **本项目解决方案** | 微信订阅消息自动提醒，支持多时间点(前3天/前1天/当天) |
| **覆盖程度** | ✅ **完全覆盖** |
| **体验提升** | 从"手动记录+易遗漏"到"自动提醒+零遗漏"，彻底解决痛点 |

#### 详细分析

**F-01 爬虫数据源失效 [致命]**

| 维度 | 分析 |
|------|------|
| 风险描述 | 院校官网作为唯一数据源，存在改版、反爬、域名变更等风险，一旦失效将导致核心功能瘫痪 |
| 潜在后果 | 夏令营信息无法更新，用户无法获取新信息，产品价值归零 |
| 发生场景 | 1. 院校官网改版，页面结构变化；2. 院校增加反爬机制(验证码、IP封禁)；3. 院校更换域名或关闭旧站点 |
| 缓解措施 | 1. 建立多数据源备份(公众号、论坛、用户上报)；2. 爬虫监控告警系统，故障即时通知；3. 模块化爬虫设计，单院校故障不影响其他；4. 建立人工录入兜底机制 |

**F-02 信息结构化提取准确率低 [高危]**

| 维度 | 分析 |
|------|------|
| 风险描述 | 公告格式千差万别，自动提取关键信息(截止日期、材料清单、申请条件)存在准确率问题 |
| 潜在后果 | 用户获取错误信息，错过截止日期或提交错误材料，产生严重后果，引发用户投诉甚至法律纠纷 |
| 发生场景 | 1. 截止日期格式多样(2026.05.01/2026年5月1日/5月1日截止)；2. 材料清单散落在不同段落；3. 申请条件使用复杂句式 |
| 缓解措施 | 1. 引入AI大模型辅助提取，提升准确率；2. 建立人工审核机制，关键信息二次确认；3. 明确免责声明，引导用户以官网为准；4. 提供"纠错反馈"入口，用户参与修正 |

**F-03 微信订阅消息配额限制 [高危]**

| 维度 | 分析 |
|------|------|
| 风险描述 | 微信订阅消息有严格的配额限制，单用户单模板只能订阅一次，且服务号有日发送总量限制 |
| 潜在后果 | 用户量大时，提醒消息无法发送，核心功能失效 |
| 发生场景 | 1. 用户订阅多个夏令营提醒，超出配额；2. 服务号日发送量超限；3. 用户未主动订阅导致无法发送 |
| 缓解措施 | 1. 引导用户多次订阅(每次设置提醒时都调用订阅接口)；2. 申请提高配额(微信认证服务号)；3. 提供小程序内提醒+微信消息双重保障；4. 重要提醒优先发送，次要提醒降级处理 |
